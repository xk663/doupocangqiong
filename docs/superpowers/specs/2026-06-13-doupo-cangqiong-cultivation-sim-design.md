# 斗破苍穹 · 修炼模拟 — 前端设计文档

**日期:** 2026-06-13  
**类型:** 纯前端原型（无后端）  
**技术栈:** 单文件 HTML/CSS/JS SPA，零依赖  
**文件名:** `index.html`

---

## 1. 产品概述

东方玄幻修炼模拟游戏 Web 前端。玩家从标题画面进入，经历创角流程后，在斗气大陆自由探索、管理修炼驻地、培养角色。

**美学:** 古卷金纹 — 深黑底 + 暗金文字(`#d4a84b`) + 异火纹路 SVG + Noto Serif SC（标题）/ Noto Sans SC（正文）

---

## 2. 页面体系（三阶段）

### 阶段一：标题画面
- 全屏暗黑背景 + 金色粒子飘动
- 中央大字「斗破苍穹」（书法体，暗金）
- 副标题「修炼模拟 · 大世界 · 自由探索」
- 菜单按钮（纵向排列）：开启新人生 / 继续人生 / 更新日志 / 设置 / 关于
- 版本号 + 版权信息

### 阶段二：创角向导（顶部步骤指示器：壹→贰→叁→肆）

**第零步：基础信息 + 难度**
- 姓名（文本输入，默认随机生成）
- 性别：男 / 女 / 扶她（三选一）
- 年龄：12-80 滑块
- 难度：无限火力(99999) / 爽文男主(200) / 简单(150) / 普通(100) / 困难(50) / 极限模式(60) / 凡人修仙(30)
- 当前可用点数实时显示（绿→黄→红渐变）

**第一步：选择种族（11族，各消耗不同点数）**
- 人族(0) / 妖族(10) / 太虚古龙族(30) / 天妖凰族(28) / 九幽地冥蟒族(22) / 蛇人族(15) / 远古天凰(32) / 远古天蛇(26) / 九彩吞天蟒(35) / 龙凰(40) / 吞灵族(38)

**第二步：选择身份（4种）**
- 散修(0) / 佣兵(5) / 世家子弟(10) / 炼药师(15)

**第三步：调整数值**
- 基础属性（可调 0-20，每点消耗1点数）：力量 / 身法 / 耐力 / 斗气 / 灵魂力 / 速度
- 表现属性（种族系数 × 基础属性，不可调）：生命上限 / 斗气上限 / 物理攻击 / 斗气攻击 / 物理防御 / 斗气防御 / 物理命中 / 斗气命中 / 暴击率

**第四步：天赋词条 + 斗气属性**
- 50 天赋词条（平庸×10 0点 / 普通×10 5点 / 稀有×10 15点 / 史诗×10 30点 / 传说×10 50点）
- 12 斗气属性：风/水/火/土/木/冰/雷/光/暗/金/毒/空间（第一属性0点，第二属性10点）

### 阶段三：游戏主界面（侧栏导航，72px 暗色侧栏）

**大世界地图（默认首页）**
- 斗气大陆 SVG/CSS 地图（势力分布 + 能量浓度）
- 顶部信息栏：坐标、纪元、斗气浓度仪表、境界
- 右下悬浮：世界事件面板 + 探索按钮
- 点击区域弹出区域详情模态框

**修炼驻地**
- 俯视图布局：乌坦城萧家后院，初始三房间
- 闭关室（聚灵阵品阶 + 修炼者）
- 炼药房（药鼎等级 + 炼丹进度）
- 武技阁（功法研习状态）
- 拓建按钮（上·洞天福地 / 下·地下密室 / 左·西厢 / 右·东厢）

**人员管理**
- 角色卡片网格（六角形头像框 + 境界光环）
- 数据：修为阶别、功法熟练度、灵魂力量、异火持有、忠诚度
- 点击角色弹出完整详情模态框

## 4. 微交互动画体系

| 交互 | 动画 | 时长 |
|------|------|------|
| 页面切换 | 淡入淡出 + 侧栏金线滑动 | 300ms ease-out |
| 模态框打开 | 缩放+淡入 + 背景模糊渐深 | 250ms spring |
| 卡片悬停 | Y轴-2px + 金边增亮 + 阴影扩散 | 200ms |
| 数据更新 | 数值跳动 + 金色微光闪过 | 400ms |
| 异火纹路 | 呼吸明暗 + 光点沿路径流动 | 3s 循环 |
| 通知弹出 | 右上滑入 + 3s停留 + 淡出 | 300ms |
| 境界突破 | 全屏金色粒子 + 屏幕震动 + 法则符文 | 1500ms |

## 5. LLM 交互设计

- **触发：** 右下角悬浮异火按钮（脉动光效）
- **面板：** 底部滑入半屏面板，暗底金边
- **对话框：** 玩家右侧金边气泡、AI 左侧银边气泡
- **上下文标签：** 显示当前所在位置
- **快捷指令：** 预设按钮组（开始修炼/查看地图/炼药/探索周边/角色状态）
- **流式输出 mock：** 逐字显示 + 光标闪烁
- **输入框：** 异火纹路聚焦动画

## 6. 设置面板

### API 配置
- API 接口地址（URL 输入）
- API Key（密码遮蔽输入）
- 模型名称
- 连接测试按钮
- 多 API 管理列表（添加/删除/设为默认）
- 当前活跃 API 标识

### 正文规则（System Rules）
- Textarea 编辑区
- 开关：启用自定义 / 使用内置默认
- 注入机制：不写回主聊天预设，由代码作独立 system 消息注入
- 关闭自定义 → 自动回退当前版本内置默认
